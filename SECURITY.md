# Security Policy

## Supported Version

Security fixes are currently provided for the latest released `phantom.py` version only.

## Reporting a Vulnerability

Please report vulnerabilities privately.

Include:

- affected version
- impact summary
- reproduction steps
- proof-of-concept (if available)
- any suggested fix

Do not open public issues for unpatched vulnerabilities.

## Scope Notes

Phantom protects message content using client-side encryption, but does not by itself hide network-level metadata (IP/timing) without Tor/VPN.

## Disclosure

- You report privately
- We validate and prepare a fix
- We publish release notes after a patch is available
